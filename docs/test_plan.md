# Test Plan

## Project Title
Student Task Tracker

## Purpose
The purpose of this test plan is to verify that the Student Task Tracker application functions correctly and that its main features work as intended.

## Features to Test
- Add task
- Delete task
- Update task
- Mark task complete
- Filter tasks
- Display all tasks

## Planned Test Cases

### Test Case 1: Add Task
**Objective:** Verify that a new task can be created and stored correctly.  
**Expected Result:** The task is added to the task list with the correct values.

### Test Case 2: Delete Task
**Objective:** Verify that an existing task can be removed.  
**Expected Result:** The correct task is removed from the task list.

### Test Case 3: Update Task
**Objective:** Verify that task details can be edited.  
**Expected Result:** The selected task reflects the updated values.

### Test Case 4: Mark Task Complete
**Objective:** Verify that a task can be marked as complete.  
**Expected Result:** The task's completion status changes to true.

### Test Case 5: Filter by Priority
**Objective:** Verify that tasks can be filtered by LOW, MEDIUM, or HIGH priority.  
**Expected Result:** Only tasks matching the selected priority are displayed.

### Test Case 6: Display All Tasks
**Objective:** Verify that all tasks are shown when requested.  
**Expected Result:** Every stored task is displayed.

## Testing Method
Testing will be done by running the program and checking whether the output matches the expected behavior for each feature.
